# Telegram QA Assistant Bot

Этот репозиторий содержит бота для Telegram, который помогает тестировщикам (QA-инженерам) быстро получать
тестовые банковские карты и файлы определённых размеров. Бот написан на Python с использованием
библиотек **pyTelegramBotAPI** и **Faker**. 

## Возможности

- Генерация тестовых номеров карт популярных платёжных систем (Visa, MasterCard, Maestro, JCB).
- Создание файлов различных форматов (`.jpg`, `.png`, `.doc`, `.txt`, `.pdf` и др.)
  заданного размера от 1 байта до 45 MB. Размер может указываться в байтах, килобайтах или мегабайтах.
- Удобное меню с кнопками: бот запрашивает только нужную информацию и позволяет вернуться к началу на каждом шаге.

## Требования

- Python 3.9+ (если запускается без Docker).
- Токен вашего бота, полученный от [@BotFather](https://t.me/BotFather) в Telegram.

## Быстрый старт

### 1. Клонирование репозитория

Склонировать репозиторий на локальную машину или развернуть на сервере:

```bash
git clone https://github.com/Reliqum/qa-assistant-bot.git
cd qa-assistant-bot
```

### 2. Настройка переменных окружения

Создать файл `.env` на основе `.env.example` и вставить туда токен вашего бота:

```env
TELEGRAM_TOKEN=ваш-токен-бота
```

### 3. Запуск без Docker

Установить зависимости и запустить бота локально:

```bash
# Установка зависимостей
pip install -r requirements.txt

# Запуск бота
python main.py
```

После запуска бот будет слушать сообщения в Telegram. Отправьте команду `/start` в чат с ботом, чтобы увидеть меню.

### 4. Запуск в Docker

Для удобного развёртывания можно использовать Docker и `docker-compose`.

1. Убедиться, что установлены Docker и Docker Compose.
2. Скопировать файл `.env.example` в `.env` и прописать свой токен:

   ```bash
   cp .env.example .env
   # затем отредактировать .env и вставить TELEGRAM_TOKEN
   ```

3. Собрать образ и запустить контейнер:

   ```bash
   docker compose up -d --build
   ```

Контейнер будет автоматически перезапускаться при сбоях (`restart: unless-stopped`).

## Структура проекта

- **main.py** — основной исходный код бота. В нём реализована логика меню и обработчики сообщений.
- **requirements.txt** — список зависимостей Python.
- **Dockerfile** — рецепт сборки Docker-образа.
- **docker-compose.yml** — конфигурация для удобного запуска с помощью Docker Compose.
- **.env.example** — пример файла конфигурации. Используйте его как шаблон для `.env`.
- **.gitignore** и **.dockerignore** — списки исключений для Git и Docker соответственно.

## Как это работает

1. Пользователь отправляет команду `/start`.
2. Бот предлагает выбрать: сгенерировать карту или файл.
3. В зависимости от выбора бот запрашивает дополнительные параметры (тип карты, расширение файла, единицы измерения и размер).
4. Бот возвращает номер тестовой карты или отправляет файл нужного размера прямо в чат.
